Cryptocurrencies APIs


Introduction


This project was thought of based on my interest on data and cryptocurrencies. This project is focused on working with the website Coin Market Cap which can be seen in the media section. The whole goal of this project was to create a code that would be able to first ask the user which cryptocurrencies they would like to know about. Then from there the data would be converted to a SQL database where the information can be viewed. This project turned out quite well, other than the input having to be entered in a specific way. The data is entered seamlessly through the whole process and then ends in the SQL database where further analysis can be conducted. Below I have added both the full python code in a py file, as well as examples for the potential SQL analysis formulas on a word document into GitHub.

Code Run Through


Starting off the code starts by asking the user to input which cryptocurrencies they would like to view or analyze. The input has to be entered in a specific way in terms of spacing and commas so that the names can then be used to get the information from Coin Market Cap online. After the input is added in, it will convert the entire list to all lowercase no matter what they enter and add a '-' in between cryptocurrencies that are multiple words. This method is to retrieve the crypto through the slug format, which is all lowercase with the '-' in between spaces. Please see below an example of an input for the code:




Seen above we can see that a long list of coins was added including Bitcoin, Ethereum, Tether, BNB ,Solana, XRP, Litecoin, Dogecoin, Toncoin, Cardano, Avalanche, Shiba Inu, Bitcoin Cash, Polkadot, Tron, and finally error coin1. I purposefully entered 'error coin1' to show the reaction of the code if a coin which is not recognized is entered or if the input is not done correctly. On the right here the ouput of what is printed to the user is shown. It first states all of the crypto currencies which the code was not able to access and then continues on to say please check spelling and formatting to ensure the coin is added into the database. Then it will print out the entire list of cryptocurrencies which have been successfully added into the SQL database. The output will complete with stating that the data has been successfully added into the data base.





Moving forward to explain the SQL portion of it. The user will have to first create a database in their respective SQL platform. In this case I have created an SQL database called 'CryptoAnalysis' and I am using Postgre for my server of choice. The first action will be to drop any existing tables with the potential name 'cryptocurrencies' and to ensure that the new table only contains relevant and the most recent data requested. Following that it will move to create a new table with the name 'cryptocurrencies' with all the related columns in a float format. This is because many of the values can be either very large or extremely small making precision hard to achieve in terms of classfiying the values. Then using the connection it will begin to insert the values that were aquired for each successfully listed cryptocurrency into the table created. This is also when it will say that the cryptocurrencie has been successfully added as seen above. Finally, the connection to the database will close or it will produce and error if the connection was unsuccessful.
